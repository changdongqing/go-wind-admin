# 修改记录（Change Records）

> 本目录承载 `thingmodel` 领域已上线/已成型设计文档之后的**变更提案**（CR, Change Request）。
> 每份 CR 是一个独立、可评审、可回滚的设计修订单元，按编号顺序追加。

## 目录约定

- 文件命名：`CR-<三位编号>-<简短标题>.md`
- 编号一经分配不复用；废弃 CR 在头部标记 `状态：已废弃` 并保留文件。
- 评审流程：草案 → 评审中 → 已通过 / 已废弃 → 已实施。
- CR 通过后，对应主线设计文档（如 `特征管理/09-*.md`）由实施同学同步修订，并在 CR 文件末尾注明"已合入文档版本"。
