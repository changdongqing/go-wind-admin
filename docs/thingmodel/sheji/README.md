## 一句话结论

本功能以**新的领域模块**形式落地（而非插件），新建 **`thingmodel` 领域域**，在其中实现 **物理量分类 + 单位 + 单位换算引擎** 三个子能力，作为整个物模型体系的**基础字典层**，为后续物模型属性（Property）定义、设备数据上报、跨单位归一化提供支撑。

> 关于"模块还是插件"的判定，见 [`01-架构决策.md`](./01-架构决策.md)。

## 设计目标

1. **完整覆盖单位清单**：承载《物模型属性单位清单》中 40+ 物理量分类、220+ 单位，含符号、基准单位、换算关系。
2. **可换算**：除对数单位/无量纲/计数单位外，绝大多数单位可在同一物理量内双向换算，并提供换算 API。
3. **可治理**：单位作为物模型属性的基础字典，支持启用/停用、排序、租户隔离、引用约束（被属性引用后不可物理删除）。
4. **零侵入**：完全复用现有 Contract-First 流水线（proto → `make api` → `make ts`）与三层架构，不引入新框架、新依赖。
5. **可扩展**：为后续物模型属性定义（`thing_property`）、设备实时数据归一化、单位偏好设置预留挂钩点。

## 关键技术决策摘要

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 落地形态 | **领域模块**（非插件） | 全项目无任何插件化机制；后端按领域域模块化、前端按 router module 模块化 |
| 领域归属 | 新建 `thingmodel` 领域域 | 单位是物模型属性的基础数据，与未来 Property/Event/Service 同域 |
| 数据结构 | **物理量分类（1）→ 单位（N）** 双表 | 镜像 `dict_type` + `dict_entry`，天然一对多；分类即"基准单位容器" |
| 换算模型 | `y = x · factor + offset`（仿射）+ 类型标记 | 覆盖 99% 可换算单位；对数/不可换算/条件换算用类型枚举区分 |
| 表达式存储 | 系数字段优先，`formula_expr` 仅作展示 | 避免 `eval` 注入风险；可换算单位一律走结构化系数 |
| 国际化 | 单位符号全球通用；名称走 i18n | 复用 `language` + i18n 子表模式（参照 dict_entry_i18n） |
| 前端布局 | `Splitter` 左右分栏 | 复用 dict 页面范式：左物理量分类树/表，右单位列表 |
