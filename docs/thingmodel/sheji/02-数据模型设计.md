# 02 · 数据模型设计

> 本章定义"模型属性单位管理"的领域模型、ER 关系、数据库表结构（字段 / 索引 / 约束）。
> 表结构风格严格对齐现有 `dict_type` / `dict_entry`（见 `internal/data/ent/schema/dict_*.go`），保证团队认知一致。

---

## 1. 领域模型

```
┌───────────────────────────┐         ┌──────────────────────────────────┐
│   UnitCategory            │ 1     N │   Unit                           │
│   (物理量分类)             │─────────│   (单位)                          │
│                           │ 属于    │                                  │
│ • code (temperature...)   │         │ • code (celsius / kelvin)        │
│ • name (温度)              │         │ • symbol (℃ / K)                 │
│ • quantity (温度量)        │         │ • is_base (基准单位标记)          │
│ • icon                    │         │ • conversion_type                │
│                           │         │ • factor / offset                │
│                           │         │ • formula_expr (展示用)           │
└───────────────────────────┘         │ • precision / is_si / is_legal   │
                                      │ • reference_count (预留)          │
                                      └──────────────────────────────────┘
```

**关系**：一个物理量分类（UnitCategory）下挂多个单位（Unit），呈一对多。同一分类内的单位以"基准单位"为锚点互相换算（详见 [03 换算引擎](./03-单位换算引擎设计.md)）。

### 1.1 关键设计抉择

| 抉择点 | 方案 | 理由 |
|--------|------|------|
| 基准单位如何表达 | `Unit.is_base = true` 标记，**不**在 Category 上存 `base_unit_id` | 避免"Unit→Category→Unit"循环外键；基准单位通过 `where category_id=? and is_base=true` 查得；与清单"基准单位"列语义一致 |
| i18n 是否独立子表 | **不建** `unit_i18n` 子表，主表存 `name`(中) + `name_en`(英) | 单位为国际标准，符号全球通用，名称仅需中英双语；建子表过度设计。（dict_entry 建 i18n 是因其业务条目需多语且可变，单位无此需求） |
| 换算公式如何存 | 结构化 `factor` + `offset` 数值字段，`formula_expr` 仅作展示 | 杜绝 `eval` 注入；可在 DB 层聚合/校验；99% 可换算单位可被 `y=x·f+o` 覆盖 |
| 引用计数 | `reference_count` 字段预留，本期恒为 0，由未来 `thing_property` 维护 | 为"被引用禁止删除"预留；本期不实现引用方，字段先占位 |
| 多租户 | 沿用 `tenant_id`（系统预置 `tenant_id=0`） | 与 dict 等全表一致；租户可扩展私有单位 |

## 2. 表结构

### 2.1 `thingmodel_unit_categories`（物理量分类表）

> 对应清单"属性分类"列：温度、压力、长度、面积、体积、质量、时间、速度……

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | uint32 | PK, auto_increment | 来自 `mixin.AutoIncrementId` |
| `code` | string | NOT NULL, UNIQUE(租户内), immutable | 物理量编码，如 `temperature` / `pressure` / `length` |
| `name` | string | NOT NULL | 中文名，如"温度""压力" |
| `name_en` | string | optional | 英文名，如 "Temperature" |
| `quantity` | string | optional | 量纲/物理量名，如"热力学温度""力"（对应 SI 基本量） |
| `base_unit_symbol` | string | optional | 基准单位符号冗余（如"K""Pa""m"），仅展示，权威值在 Unit 表 |
| `icon` | string | optional | Iconify 图标名，前端分类展示用 |
| `description` | string | optional | 描述/备注 |
| `is_enabled` | bool | default true | 来自 `mixin.IsEnabled`，停用后该分类下单位不出现在属性单位选择器 |
| `sort_order` | uint32 | default 0 | 来自 `mixin.SortOrder` |
| `tenant_id` | uint32 | default 0 | 来自 `mixin.TenantID`，0=系统全局 |
| `created_by/updated_by/deleted_by` | uint32 | optional | 来自 `mixin.OperatorID` |
| `created_at/updated_at/deleted_at` | timestamp | optional | 来自 `mixin.TimeAt` |

**Ent Schema 骨架**（风格对齐 `dict_type.go`）：

```go
// thingmodel_unit_categories 物理量分类表
type UnitCategory struct{ ent.Schema }

func (UnitCategory) Annotations() []schema.Annotation {
    return []schema.Annotation{
        entsql.Annotation{Table: "thingmodel_unit_categories", Charset: "utf8mb4", Collation: "utf8mb4_bin"},
        entsql.WithComments(true),
        schema.Comment("物模型-物理量分类表"),
    }
}

func (UnitCategory) Fields() []ent.Field {
    return []ent.Field{
        field.String("code").Comment("物理量编码，如 temperature/pressure").NotEmpty().Immutable().Optional().Nillable(),
        field.String("name").Comment("中文名，如 温度").NotEmpty().Optional().Nillable(),
        field.String("name_en").Comment("英文名").Optional().Nillable(),
        field.String("quantity").Comment("量纲/物理量名").Optional().Nillable(),
        field.String("base_unit_symbol").Comment("基准单位符号（冗余展示）").Optional().Nillable(),
        field.String("icon").Comment("Iconify 图标名").Optional().Nillable(),
        field.String("description").Comment("描述").Optional().Nillable(),
    }
}

func (UnitCategory) Mixin() []ent.Mixin {
    return []ent.Mixin{
        mixin.AutoIncrementId{}, mixin.TimeAt{}, mixin.OperatorID{},
        mixin.IsEnabled{}, mixin.SortOrder{}, mixin.TenantID[uint32]{},
    }
}

func (UnitCategory) Edges() []ent.Edge {
    return []ent.Edge{
        edge.To("units", Unit.Type).Required().
            Annotations(entsql.Annotation{OnDelete: entsql.Cascade}).
            StorageKey(edge.Column("category_id")),
    }
}
```

**索引**（命名对齐 dict：`uix_` 唯一 / `idx_` 普通）：

```go
func (UnitCategory) Indexes() []ent.Index {
    return []ent.Index{
        // 租户级唯一：同一租户下 code 唯一
        index.Fields("tenant_id", "code").Unique().StorageKey("uix_thingmodel_unit_cat_tenant_code"),
        index.Fields("tenant_id").StorageKey("idx_thingmodel_unit_cat_tenant_id"),
        index.Fields("is_enabled").StorageKey("idx_thingmodel_unit_cat_is_enabled"),
        index.Fields("sort_order").StorageKey("idx_thingmodel_unit_cat_sort_order"),
    }
}
```

### 2.2 `thingmodel_units`（单位表）

> 对应清单每一行单位：开尔文 K、摄氏度 ℃、千帕 kPa……

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | uint32 | PK | |
| `category_id` | uint32 | NOT NULL, FK→UnitCategory | 所属物理量分类 |
| `code` | string | NOT NULL, UNIQUE(租户内) | 单位编码，如 `celsius` / `kelvin` / `kilopascal`（程序引用用） |
| `symbol` | string | NOT NULL | 单位符号，如 `℃` `K` `kPa` `m³/h`（展示用） |
| `name` | string | NOT NULL | 中文名，如"摄氏度""千帕" |
| `name_en` | string | optional | 英文名，如 "Celsius" |
| `is_base` | bool | default false | **是否基准单位**（每分类有且仅有一个 true） |
| `conversion_type` | enum | NOT NULL | 换算类型：`LINEAR` / `AFFINE` / `LOGARITHMIC` / `CONDITIONAL` / `NONE`（见 [03](./03-单位换算引擎设计.md)） |
| `factor` | double | default 1.0 | 线性系数 `k`（正向：基准 = 原值 · factor + offset） |
| `offset` | double | default 0.0 | 偏移量 `b`（仅 AFFINE 非 0，如温度） |
| `formula_expr` | string | optional | 人类可读公式/说明，仅展示，如 `10^(x/10) mW`、`x+273.15`、`依赖分子量 M 与温度 T` |
| `precision` | int | optional, default 2 | 建议显示精度（小数位） |
| `is_si_unit` | bool | default false | 是否 SI 单位（含 SI 词头组合） |
| `is_legal_unit` | bool | default false | 是否中国法定计量单位（GB 3100-1993） |
| `reference_count` | uint32 | default 0 | 被物模型属性引用次数（**预留**，本期恒 0） |
| `is_enabled` | bool | default true | |
| `sort_order` | uint32 | default 0 | 基准单位建议 sort_order=0 排首位 |
| `tenant_id` | uint32 | default 0 | |
| 审计字段 | — | — | operator/time mixin |

**Ent Schema 骨架**：

```go
type Unit struct{ ent.Schema }

func (Unit) Annotations() []schema.Annotation {
    return []schema.Annotation{
        entsql.Annotation{Table: "thingmodel_units", Charset: "utf8mb4", Collation: "utf8mb4_bin"},
        entsql.WithComments(true),
        schema.Comment("物模型-单位表"),
    }
}

func (Unit) Fields() []ent.Field {
    return []ent.Field{
        field.Uint32("category_id").Comment("所属物理量分类 ID").Nillable(),
        field.String("code").Comment("单位编码，如 celsius").NotEmpty().Optional().Nillable(),
        field.String("symbol").Comment("单位符号，如 ℃").NotEmpty().Optional().Nillable(),
        field.String("name").Comment("中文名").NotEmpty().Optional().Nillable(),
        field.String("name_en").Comment("英文名").Optional().Nillable(),

        field.Bool("is_base").Comment("是否基准单位").Default(false).Nillable(),

        field.Enum("conversion_type").
            NamedValues(
                "Linear", "LINEAR",
                "Affine", "AFFINE",
                "Logarithmic", "LOGARITHMIC",
                "Conditional", "CONDITIONAL",
                "None", "NONE",
            ).
            Default("LINEAR").
            Comment("换算类型"),

        field.Float64("factor").Comment("线性系数 k（基准=原值·k+offset）").Default(1.0).Nillable(),
        field.Float64("offset").Comment("偏移量 b（仅仿射非0）").Default(0.0).Nillable(),
        field.String("formula_expr").Comment("公式说明（仅展示）").Optional().Nillable(),

        field.Int("precision").Comment("建议显示精度（小数位）").Default(2).Optional().Nillable(),
        field.Bool("is_si_unit").Comment("是否 SI 单位").Default(false).Nillable(),
        field.Bool("is_legal_unit").Comment("是否中国法定计量单位").Default(false).Nillable(),
        field.Uint32("reference_count").Comment("被物模型属性引用次数（预留）").Default(0).Nillable(),
    }
}

func (Unit) Mixin() []ent.Mixin {
    return []ent.Mixin{
        mixin.AutoIncrementId{}, mixin.TimeAt{}, mixin.OperatorID{},
        mixin.SortOrder{}, mixin.IsEnabled{}, mixin.TenantID[uint32]{},
    }
}

func (Unit) Edges() []ent.Edge {
    return []ent.Edge{
        edge.From("category", UnitCategory.Type).Ref("units").Unique(),
    }
}
```

**索引**：

```go
func (Unit) Indexes() []ent.Index {
    return []ent.Index{
        // 租户级唯一：同一租户下 code 唯一（程序引用的稳定标识）
        index.Fields("tenant_id", "code").Unique().StorageKey("uix_thingmodel_unit_tenant_code"),

        // 按分类查询单位（最高频查询：属性选单位时拉某分类下全部单位）
        index.Fields("tenant_id", "category_id").StorageKey("idx_thingmodel_unit_tenant_category"),
        index.Fields("category_id").StorageKey("idx_thingmodel_unit_category_id"),

        // 基准单位定位（每分类查 is_base=true）
        index.Fields("category_id", "is_base").StorageKey("idx_thingmodel_unit_category_base"),

        // 常用筛选
        index.Fields("is_enabled").StorageKey("idx_thingmodel_unit_is_enabled"),
        index.Fields("conversion_type").StorageKey("idx_thingmodel_unit_conversion_type"),
        index.Fields("tenant_id").StorageKey("idx_thingmodel_unit_tenant_id"),

        // 符号/名称模糊搜索
        index.Fields("symbol").StorageKey("idx_thingmodel_unit_symbol"),
    }
}
```

## 3. 约束（应用层 + DB 层）

部分约束无法用纯 schema 表达，需在 Service 层校验（[05 后端实现](./05-后端实现设计.md) 的 Service 校验小节）：

| 编号 | 约束 | 实现层 | 说明 |
|------|------|--------|------|
| C1 | 每个分类有且仅有一个 `is_base=true` 的单位 | Service + 部分唯一索引 | 建议在迁移脚本补 partial unique index：`WHERE is_base = true` 按 `(tenant_id, category_id)` 唯一 |
| C2 | `is_base=true` 的单位 `factor=1` 且 `offset=0` | Service | 基准单位定义如此 |
| C3 | `conversion_type=LINEAR` 时 `offset` 必须为 0 | Service | 纯线性无偏移 |
| C4 | `conversion_type=NONE` 的单位不可作为换算源/目标 | 换算 API 运行期 | 见 [03](./03-单位换算引擎设计.md) |
| C5 | 删除分类时，其下单位级联删除（`OnDelete: Cascade`） | DB | 已在 edge 注解声明 |
| C6 | 删除单位时，若 `reference_count > 0` 则禁止物理删除，仅允许 `is_enabled=false` | Service | 为未来 thing_property 引用预留 |
| C7 | `code` 不可变（`Immutable()`），`code` 租户内唯一 | DB + ent | 程序引用的稳定标识 |
| C8 | `factor` 不可为 0（基准单位 `factor=1`） | Service | 避免除零；换算反向需除以 factor |

> **关于 C1 的 partial unique index**：Ent schema 层难以直接表达"每组 category 仅一个 is_base"的偏唯一约束。推荐做法：在生成的迁移 SQL 之后，手动追加（或在 `data/migrate` 脚本中）：
> ```sql
> CREATE UNIQUE INDEX uix_thingmodel_unit_base
>   ON thingmodel_units (tenant_id, category_id) WHERE is_base = true;
> ```
> 并在 Service 层做"切换基准单位"事务：旧基准置 false → 新基准置 true，保证任一时刻唯一。

## 4. 数据示例（取自清单）

**物理量分类**（节选）：

| code | name | base_unit_symbol | quantity |
|------|------|------------------|----------|
| temperature | 温度 | K | 热力学温度 |
| pressure | 压力 | Pa | 压强 |
| length | 长度 | m | 长度 |
| energy | 能量 | J | 能、功、热 |
| frequency | 频率 | Hz | 频率 |
| concentration | 浓度 | ppm | 物质的量分数（混合） |
| count | 计数 | — | 无量纲计数 |

**温度分类下的单位**（完整展示换算字段语义）：

| code | symbol | name | is_base | conversion_type | factor | offset | formula_expr |
|------|--------|------|---------|-----------------|--------|--------|--------------|
| kelvin | K | 开尔文 | **true** | LINEAR | 1.0 | 0.0 | x*1 |
| celsius | ℃ | 摄氏度 | false | AFFINE | 1.0 | 273.15 | x+273.15 |
| fahrenheit | ℉ | 华氏度 | false | AFFINE | 5/9 (0.5556) | 255.372 | (x+459.67)*5/9 |
| rankine | °R | 兰氏度 | false | LINEAR | 5/9 (0.5556) | 0.0 | x*5/9 |

> 验证：50℃ → 基准 K = 50×1.0 + 273.15 = **323.15 K** ✓（与清单 `x+273.15` 一致）
> 验证：50℉ → 基准 K = 50×(5/9) + 255.372 = 27.778 + 255.372 = **283.15 K** ✓（=(50+459.67)×5/9）

完整的 40+ 分类与 220+ 单位规划见 [07 种子数据与实施计划](./07-种子数据与实施计划.md)。
