# 05 · 前端实现设计

> 本章定义三个前端（react / vue-element / vue-vben）共享的页面形态、组件结构、Hook/Composable、路由、i18n、权限码。
> 严格遵循项目"Shared Frontend Data-Layer Pattern"（[CLAUDE.md §Shared Frontend Data-Layer Pattern]）。

**字段权限**：
- LOCAL 来源：所有 spec 字段可编辑（含 dataType/access_mode 等结构字段）
- DEFAULT/GLOBAL 来源：仅白名单字段可编辑；其他字段灰显并标 ⓘ
- 产品 `status=PUBLISHED` 时：整个 Drawer 只读 + 顶部黄条提示

## 2. 数据层 — Hook / Composable

按"Shared Frontend Data-Layer Pattern"，三个前端的 hook 命名与查询键完全一致。

### 2.1 React (`api/hooks/`)

```ts
// useThingModelCategoryDefaultFeatures.ts
export function useListCategoryDefaultFeatures(params: ListCategoryDefaultFeaturesRequest) {
  return useQuery({
    queryKey: ['listCategoryDefaultFeatures', params],
    queryFn: () => apiClient.thingModelCategoryDefaultFeatureService.List(params),
    enabled: !!params.categoryId,
  });
}

export function useBatchAddCategoryDefaultFeatures() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (req: BatchAddCategoryDefaultFeaturesRequest) =>
      apiClient.thingModelCategoryDefaultFeatureService.BatchAdd(req),
    onSuccess: (_, vars) => {
      qc.invalidateQueries({ queryKey: ['listCategoryDefaultFeatures'] });
    },
  });
}

// useThingModelProduct.ts
export function useListProducts(query: PaginationQuery<ListProductsRequest>) {
  return useQuery({
    queryKey: ['listProducts', query],
    queryFn: () => apiClient.thingModelProductService.List(query.toRawParams()),
  });
}
export function useCreateProduct() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (req: CreateProductRequest) =>
      apiClient.thingModelProductService.Create({ data: req }),
    onSuccess: () => qc.invalidateQueries({ queryKey: ['listProducts'] }),
  });
}
export function useUpdateProduct() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (req: UpdateProductRequest) => {
      const mask = makeUpdateMask(req);
      return apiClient.thingModelProductService.Update({
        product: req,
        updateMask: mask,
      });
    },
    onSuccess: (_, vars) => {
      qc.invalidateQueries({ queryKey: ['listProducts'] });
      qc.invalidateQueries({ queryKey: ['getProduct', vars.id] });
    },
  });
}

// useThingModelProductFeature.ts
export function usePullFromDefault() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (req: PullFromDefaultRequest) =>
      apiClient.thingModelProductFeatureService.PullFromDefault(req),
    onSuccess: (_, vars) => {
      qc.invalidateQueries({ queryKey: ['listProductFeatures', { productId: vars.productId }] });
    },
  });
}
```

非组件代码（store / route guard）使用 `fetchListProducts` 等纯函数版本。

### 2.2 Vue (vue-element / vue-vben) `api/composables/`

```ts
// useThingModelProduct.ts
export function useListProducts(query: Ref<PaginationQuery<ListProductsRequest>>) {
  return useQuery({
    queryKey: computed(() => ['listProducts', query.value]),
    queryFn: () => apiClient.thingModelProductService.List(query.value.toRawParams()),
  });
}
// ... 其余与 React 形态一致
```

## 3. 路由 / 菜单 / 权限

### 3.1 路由

```ts
// React
{
  path: '/thingmodel/product',
  meta: { title: 'menu.thingmodel.product', authority: ['thingmodel:product:view'] },
  children: [
    { index: true, element: <ProductListPage /> },
    { path: ':id', element: <ProductDetailPage /> },
  ],
}

// Vue (相同结构，按各自路由 DSL)
```

### 3.2 菜单

由后端 `DefaultMenus` 注入；前端不需要硬编码。三个前端的菜单都从 `/api/admin/menu/me` 取活动菜单树。

### 3.3 权限码

```
thingmodel:product:view     ← /thingmodel/product 路由 meta.authority
thingmodel:product:create   ← "+ 新增产品" 按钮
thingmodel:product:edit     ← 编辑、PullFromDefault、Publish
thingmodel:product:delete   ← 删除产品
thingmodel:category:edit    ← 分类默认模型 Drawer 内所有写操作（复用现有码）
```

## 5. 组件分解

### 5.1 公共组件

| 组件 | 路径 | 职责 |
|------|------|------|
| `<FeaturePicker />` | `components/thingmodel/` | 全局特征库选择器（Modal，按类型 Tab + 搜索 + 多选） |

## 6. 关键交互细节

### 6.1 已覆写字段的视觉标识

```
取值范围:  min[5] max[12] step[0.1]   ← 蓝色边框 + 小标记"已覆写"
单位:      [℃ ▾]                      ← 普通灰色，无覆写
```

UI 计算：对每个表单字段比较 `effective_spec[field]` 与 `feature_snapshot[field]` — 不同则标为已覆写。

### 6.2 重置覆写

二级 Drawer 上方 `[重置覆写]` 按钮：清空 `override_spec` 整体，恢复为 `feature_snapshot` 原值。
单字段重置：长按字段 → `↺ 重置此项`（仅 React 实现；Vue 端先做整体重置）。

### 6.3 PublishProduct 二次确认

```
┌─── 发布产品 ───────────────────────────────────┐
│ 发布后将冻结物模型结构：                          │
│  • 不能再增/删特征条目                           │
│  • 不能再修改特征结构（仅可调整覆写）             │
│                                                  │
│ 当前模型 12 条特征 · 4 属性 · 3 事件 · 4 服务 · 1 关系 │
│                                                  │
│              [取消]   [确认发布]                 │
└──────────────────────────────────────────────────┘
```

### 6.4 List 页的分类列展示

产品列表的"分类"列显示 `category_name (category_code)`；级联筛选框选大类时 `includeDescendants=true`。

## 7. 三前端差异点

| 维度 | React | vue-element | vue-vben |
|------|-------|-------------|----------|
| 表单 | AntDesign v6 `Form` | Element Plus `el-form` | Antd Vue `a-form` |
| Drawer | `Drawer` | `el-drawer` | `a-drawer` |
| Steps | `Steps` | `el-steps` | `a-steps` |
| Modal | `Modal` | `el-dialog` | `a-modal` |
| 树/级联 | `Cascader` | `el-cascader` | `a-cascader` |
| 表格 | `ProTable` | `el-table-v2` | `vxe-table` 或 vben 包装 |

各前端的实现遵循各自 `frontend/admin/<framework>/CLAUDE.md` 的范式，**不引入新依赖**。
