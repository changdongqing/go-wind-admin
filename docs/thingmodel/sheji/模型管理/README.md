## 一句话结论

本功能以**两块入口、两套表**的形式补齐物模型基础字典层的最后一块：

- **入口①「分类默认模型」**：藏在分类管理 level=4 细类的行内操作（Drawer），承担"**分类 → 默认特征集合**"的字典化绑定。一类一模、一模 N 特征。
- **入口②「产品管理」**（新菜单 `/thingmodel/product`）：先选 level=4 细类、再创建产品、按需从该细类的默认模型**勾选特征值复制**为产品自有模型；产品支持加全局特征、加产品本地特征、对任意来源做轻量覆写。

> "模型管理"作为**逻辑概念**贯穿两个入口；不单独建一级菜单。设计意图见 [`01-架构决策.md`](./01-架构决策.md)。

## 设计目标

1. **闭环物模型**：与已有"单位 / 特征 / 分类"串联成完整链路 — 单位 ← 特征 ← 分类默认模型 ← 产品模型 ← （未来）设备实例。
2. **快速建模**：从分类默认模型**逐个特征勾选** + 值复制到产品模型，让 90% 的常规产品在 5 分钟内完成建模。
3. **可治理**：产品 code/category_id 唯一约束、覆写白名单、`reference_count` 真正落地（终结单位管理预留的挂钩点）。
4. **零侵入**：完全复用 Contract-First 流水线（proto → `make api` → `make ts`）、三层架构、`thingmodel` 域，不引入新框架。
5. **可扩展**：本期不做版本，但保留 `status: DRAFT/PUBLISHED` 字段；未来"版本号、设备实例引用"等通过追加字段实现，零迁移。

## 关键技术决策摘要

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 落地形态 | **无独立菜单**，分两个入口（分类管理行内 Drawer + 产品管理新菜单） | 用户明确："本不存在'模型管理'菜单" |
| 产品-分类关系 | **单细类归属**（一个产品挂一个 level=4 节点） | 对齐阿里云/华为云 IoT "产品 = 设备品类"主流范式 |
| 默认模型多重性 | **一类一模**（一个 level=4 节点至多 1 个默认模型） | 与"分类即类型"语义一致；YAGNI |
| 默认模型可配层级 | **仅 level=4**（细类） | 上层为分组用途；细类是真正的"可实例化品类" |
| 默认模型 kind 范围 | **所有 kind 都可配**（含未来扩展的 kind） | 不收窄；产品本期仅允许挂 FACILITY，未来可放开 |
| 拉取继承语义 | **值复制（深拷贝）** | 产品自洽；默认模型升级不"暗中"改产品 |
| 拉取粒度 | **逐个特征勾选**（默认全选） | 对齐阿里云"从品类拷贝"形态 |
| 覆写范围 | **轻量约束白名单**（constraints/unit/defaultValue/displayName/description/sort/enabled） | 防止结构性差异；改 dataType 应"另起特征"而非"覆写" |
| 产品本地特征 | **允许**（不写回全局特征库） | 满足厂家私有功能场景，不污染全局字典 |
| 产品唯一性 | **(tenant, code) 唯一** + **(tenant, category_id, name) 唯一** | code 程序用、name+category 人读用，双重防重 |
| 表结构 | **双表拆分**：`category_default_features` 关联表 + `products`/`product_features` 产品两表 | 三层架构语义最清晰；不引入 `model` 抽象避免过度设计 |
| 产品版本 | **本期不做**，保留 `status: DRAFT/PUBLISHED` 字段 | 未来加 version 列零迁移 |
| Relation 类型 | **包含**（默认模型与产品模型都可含 RELATION 特征） | 本体语义完整；source/target 复用 spec JSON |
| `product_feature.source` | **三类**：DEFAULT / GLOBAL / LOCAL | 区分来源便于 reference_count 维护、UI 标注、未来"升级提示"；预留 INHERITED 等扩展 |
| 制造商/型号字段 | **加，optional** | 工程实际常用，UI 提示填写但不强制 |
| 产品菜单位置 | **物模型一级菜单下子菜单** `/thingmodel/product` | 与 unit/feature/category 同级，组织自洽 |
| 前端 | 分类管理：行内"配置默认模型" Drawer<br>产品管理：ProTable + 两步创建向导 + Tab 化模型编辑 Drawer | 复用项目现有 Drawer / Steps / ProTable 范式 |

## 非目标（本期不做）

- **设备实例**（`device_instance`）— 产品本期到"建模完成"为止，不涉及具体设备的实例化与数据上报。
- **产品模型版本号** — 本期 `status: DRAFT/PUBLISHED` 即足够；未来加 version 列零迁移。
- **默认模型升级广播** — 默认模型变更不"自动"通知已建产品；未来通过"升级提示"UI 引导用户主动拉取。
- **跨租户/跨分类的产品克隆** — 仅同租户内复制；克隆 LOCAL 特征跟随。
- **批量导入** — 产品/默认模型批量 Excel 导入留待下期。
- **物模型 JSON 出码（OTA 物模型下发）** — 留到设备运行时一并设计。
- **产品级"快速联调"或"模拟器"** — 物模型层不做。
