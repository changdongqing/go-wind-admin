## 2. Hooks 层（`api/hooks/feature.ts`）

镜像 `unit.ts`，类型从 `@/api/generated/admin/service/v1` 导入（`make ts` 后生成 `featureservicev1_*`）：

```typescript
import { useMutation, useQuery, type UseMutationOptions, type UseQueryOptions } from '@tanstack/react-query';
import {
  type featureservicev1_Feature,
  type featureservicev1_ListFeatureResponse,
  type featureservicev1_GetFeatureRequest,
  type featureservicev1_CreateFeatureRequest,
  type featureservicev1_DeleteFeatureRequest,
  type featureservicev1_ListFeatureByTypeRequest,
  type featureservicev1_ValidateFeatureSpecRequest,
  type featureservicev1_ValidateFeatureSpecResponse,
  type featureservicev1_FeatureType,
} from '@/api/generated/admin/service/v1';
import { makeUpdateMask, type PaginationQuery } from '@/core/transport/rest';
import { queryClient } from '@/core';
import { apiClient } from '@/api/client';

// ========== 特征列表 ==========
export function useListFeatures(q: PaginationQuery, o?: UseQueryOptions<featureservicev1_ListFeatureResponse, Error>) {
  return useQuery({ queryKey: ['listFeatures', q], queryFn: () => apiClient.featureService.List(q.toRawParams()), ...o });
}
export async function fetchListFeatures(params: PaginationQuery) {
  return queryClient.fetchQuery({ queryKey: ['listFeatures', params], queryFn: () => apiClient.featureService.List(params.toRawParams()), retry: 0 });
}

// 按类型查询（左侧树联动）
export function useListFeaturesByType(req: featureservicev1_ListFeatureByTypeRequest, o?: UseQueryOptions<featureservicev1_ListFeatureResponse, Error>) {
  return useQuery({ queryKey: ['listFeaturesByType', req], queryFn: () => apiClient.featureService.ListByType(req), ...o });
}

// ========== 特征详情 ==========
export function useGetFeature(req: featureservicev1_GetFeatureRequest, o?: UseQueryOptions<featureservicev1_Feature, Error>) {
  return useQuery({ queryKey: ['getFeature', req], queryFn: () => apiClient.featureService.Get(req), enabled: !!req.id, ...o });
}

// ========== 增删改 ==========
export function useCreateFeature(o?: UseMutationOptions<{}, Error, featureservicev1_CreateFeatureRequest>) {
  return useMutation({ mutationFn: (data) => apiClient.featureService.Create(data), ...o });
}
export function useUpdateFeature(o?: UseMutationOptions<{}, Error, { id: number; values: Record<string, any> }) {
  return useMutation({
    mutationFn: ({ id, values }) => apiClient.featureService.Update({ id, data: { ...values } as any, updateMask: makeUpdateMask(Object.keys(values ?? {})) }),
    ...o,
  });
}
export function useDeleteFeature(o?: UseMutationOptions<{}, Error, featureservicev1_DeleteFeatureRequest>) {
  return useMutation({ mutationFn: (data) => apiClient.featureService.Delete(data), ...o });
}

// ========== spec 校验（前端实时校验）==========
export function useValidateSpec(o?: UseMutationOptions<featureservicev1_ValidateFeatureSpecResponse, Error, featureservicev1_ValidateFeatureSpecRequest>) {
  return useMutation({ mutationFn: (data) => apiClient.featureService.ValidateSpec(data), ...o });
}
```

> **`apiClient.featureService`** getter 需在 `src/api/client.ts` 按现有懒加载模式新增（生成代码已提供 client 类）。

## 3. 页面布局（`index.tsx`）

镜像单位管理 `unit/index.tsx` 的 `Splitter` 左右分栏：

```tsx
import { Splitter } from 'antd';
import { useState } from 'react';
import ContentContainer from '@/layouts/components/PageContainer/ContentContainer';
import FeatureTypeTree from './FeatureTypeTree';
import FeatureList from './FeatureList';

const FeatureManagement = () => {
  const [currentType, setCurrentType] = useState<FeatureType>('PROPERTY');
  return (
    <ContentContainer heightMode="fixed" padding="16px" bottomMargin={0}>
      <Splitter style={{ height: '100%', flex: 1, minHeight: 0 }}>
        <Splitter.Panel collapsible defaultSize="28%" min="20%" max="40%" style={{ display: 'flex', flexDirection: 'column' }}>
          <FeatureTypeTree currentType={currentType} onTypeSelect={setCurrentType} />
        </Splitter.Panel>
        <Splitter.Panel style={{ display: 'flex', flexDirection: 'column' }}>
          <FeatureList featureType={currentType} />
        </Splitter.Panel>
      </Splitter>
    </ContentContainer>
  );
};
export default FeatureManagement;
```

### 3.1 `FeatureTypeTree.tsx`（左：特征类型树）

四个固定节点（非动态数据，类型本身是枚举）：

```
📁 特征类型
├── 🟦 属性 (Property)     —— 统计数 N
├── 🟧 事件 (Event)        —— 统计数 N
├── 🟩 服务 (Service)      —— 统计数 N
└── 🟪 关系 (Relation)     —— 统计数 N
```

- 选中节点高亮（镜像单位管理的选中蓝条），`onTypeSelect` 回传 `FeatureType`。
- 节点后显示该类型特征总数（可选，调 Count 接口）。
- 可折叠为"全部"，显示所有类型特征。

> 为什么用 Tree 而非 Tabs：Tree 与单位管理左侧"物理量分类列表"范式一致（Splitter 左右分栏），且未来可在类型节点下挂设备大类子节点（暖通/电气），扩展性优于 Tabs。

### 3.2 `FeatureList.tsx`（右：特征列表）

用 ProTable，**列随 `featureType` 动态变化**：

| 列 | 所有类型 | property | event | service | relation |
|----|:---:|:---:|:---:|:---:|:---:|
| code | ✓ | | | | |
| name | ✓ | | | | |
| identifier | ✓ | | | | |
| 数据类型 | | ✓ | | | |
| 访问模式 | | ✓ | | | |
| 事件级别 | | | ✓ | | |
| 调用模式 | | | | ✓ | |
| 关系类型 | | | | | ✓ |
| source→target | | | | | ✓ |
| 适用范围 | ✓ | | | | |
| 状态 | ✓ | | | | |
| 操作 | ✓ | | | | |

- `useEffect([featureType])` 触发 `actionRef.reload()`。
- `request` 走 `fetchListFeatures(new PaginationQuery({...formValues, featureType}))`。
- `toolBarRender` 新建按钮（`useAccess().hasAccessByCodes(['thingmodel:feature:create'])`）。
- 删除用 `Popconfirm` + `useDeleteFeature`，被 relation 引用时后端返回 `FEATURE_IN_USE_CANNOT_DELETE`，前端 toast 提示。

### 3.3 `FeatureDrawer.tsx`（特征新建/编辑，核心动态表单）

用 `DrawerForm` + `formRef`（无 `useForm`，遵循 react CLAUDE.md 注意事项）。结构：

```
┌─ 公共字段区 ─────────────────────────────┐
│ 特征类型 [Property ▼]（新建时可选，编辑只读）│
│ 编码 code __________  标识符 identifier __ │
│ 中文名 __________    英文名 __________    │
│ 说明 ____________________________          │
│ 适用设备范围 ____________________          │
├─ 动态 spec 区（按 featureType 渲染）───────┤
│ <PropertySpecForm /> 或 <EventSpecForm />  │
│   或 <ServiceSpecForm />                   │
│   或 <RelationSpecForm />（特殊，见 3.4）   │
├─ 治理区 ──────────────────────────────────┤
│ 排序 __  启用 [✓]                          │
└───────────────────────────────────────────┘
```

- 顶部 `feature_type` Select 切换时，动态区卸载旧组件、加载新组件。
- `onChange` 时调用 `useValidateSpec`（debounce 500ms）做后端校验，展示 `errors`。
- 提交前构造 `{ ...publicFields, spec: currentSpecValues }`。

### 3.4 `specs/PropertySpecForm.tsx`（属性动态表单）

最复杂的表单。按 `dataType` 动态显示约束字段：

```
数据类型 [double ▼]
访问模式 [R/RW]
类别   [runtime/measurement/setting/rated/...]
单位   [选择单位 ▼]（调 useListUnitsByCategory，复用单位管理 hook）

── 根据 dataType 条件渲染 ──
[int/float/double]：min ___ max ___ step ___ 默认值 ___
[bool]            ：0 的描述 ___ 1 的描述 ___
[enum]            ：枚举项列表（可增删行：value ___ label ___）
[text]            ：最大长度 ___
[struct]          ：子字段列表（可增删，每行是迷你 PropertySpecForm，递归）
[array]           ：最大长度 ___ 元素类型 [double ▼]（struct 时递归）
```

- **单位选择器**：调单位管理的 `useListUnitsByCategory`（已实现），选物理量分类 → 选单位，填入 `{unitId, unitCode, unitSymbol}`。
- **struct 递归**：子字段表单复用 PropertySpecForm 组件自身（`isNested` 模式，隐藏公共字段）。
- **enum 项增删**：用 `Form.List` 动态行。

### 3.5 `specs/EventSpecForm.tsx` / `ServiceSpecForm.tsx`

结构类似，用 `Form.List` 管理参数列表（outputParams / inputParams），每行是一个 ParamSpec 表单（key/dataType/unit/约束）。Service 比 Event 多 `callMode` + `inputParams` + `timeout`。

### 3.6 `specs/RelationSpecForm.tsx` + `RelationDrawer.tsx`

关系表单较特殊，独立抽屉：

```
关系类型 [derivedFrom ▼]（partOf/feeds/controls/derivedFrom/...）
基数     [oneToOne/oneToMany/manyToOne/manyToMany ▼]
有向     [✓]

源 source:
  类型 [feature/external ▼]
  └ feature 时：[选择特征 ▼]（按 identifier/code 搜索）
  └ external 时：实体类型 [device/space] + 编码 ___

目标 target:（同上）

关系属性（可选）：键值对列表
```

- source/target 选 feature 时，调 `useListFeatures` 搜索（可按类型预过滤）。
- 校验 `directional=false` 时提示"无向关系 source/target 顺序无关"。

## 5. 权限码

| 权限码 | 含义 |
|--------|------|
| `thingmodel:feature:view` | 查看特征 |
| `thingmodel:feature:create` / `:update` / `:delete` | 特征增改删 |
| `thingmodel:feature:validateSpec` | spec 校验（一般随 view 授予） |

- 路由 `meta.authority` 用 `['thingmodel:feature:view']`。
- 页面按钮用 `useAccess().hasAccessByCodes(['thingmodel:feature:create'])` 控制显隐（参照 react CLAUDE.md）。
- 超级管理员 `*:*:*` 自动放行。
- 权限码需在权限管理后台录入（permission 表 + 菜单 BUTTON 项）。

## 8. 前端注意事项（遵守 react CLAUDE.md）

1. `PaginationQuery` 必须 `new`。
2. 非组件环境（路由守卫等）只用 `fetchListFeatures`，不用 `useXxx`。
3. `DrawerForm` 用 `formRef`，无 `useForm`。
4. `ProTable` 的 `scroll.y` 用 `useProTableScrollY` 返回的像素值（镜像 unit）。
5. 硬编码文本全部走 i18n。
6. `meta.title` 用 `'routes:xxx'` 格式。
7. 禁止手改 `api/generated/`。
8. **动态表单的性能**：struct 递归嵌套层数建议限制 ≤ 3 层（前端防卡顿 + 业务够用）。
