## 一句话结论

本功能以**统一特征表（Thing Feature）**承载物模型的三维结构（属性 / 事件 / 服务）**外加本体关系（Relation）**，将其抽象为一类可治理、可复用的**特征字典**——属性、事件、服务、关系统一是"特征"的不同形态，用 `feature_type` 区分、用 `spec` JSON 承载各自的差异化约束。这是继"单位管理"之后的**物模型第二块基石**，为后续产品物模型模板、设备实例提供可组装的特征来源。

> 关于"统一表 vs 分表"的判定，见 [`08-特征架构决策.md`](./08-特征架构决策.md)。

## 设计目标

1. **统一抽象，差异收敛**：属性 / 事件 / 服务 / 关系四类实体统一为 `thing_feature`，公共字段（code/identifier/name/分类/排序/启用/租户）一致，差异收敛到 `spec` JSON。一次表结构、四套校验逻辑。
2. **完整覆盖特征清单**：承载《物模型特征清单》定义的通用属性（7 类）、通用事件（8 项）、通用服务（8 项）、各设备专属属性（120+）、事件（30+）、服务（25+），以及额定特征（铭牌参数 100+）。
3. **与单位管理无缝衔接**：属性特征引用 `thingmodel_unit.id`（外键），完成单位管理的"预留挂钩点"——`unit.reference_count` 由本模块维护增减。
4. **关系即特征**：本体关系（Relation）作为 `feature_type=relation` 的一类特征建模，承载 source/target/relation_type/cardinality 语义，不另建独立表。
5. **零侵入**：完全复用单位管理验证过的 Contract-First 流水线与三层架构，镜像其 proto / ent / repo / service / 前端范式。

## 关键技术决策摘要

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 建模形态 | **统一特征表** `thing_feature` | 用户明确要求"属性/事件/服务/关系都是特征属性"；公共字段占比 70%+，差异收敛到 `spec` JSON 最经济 |
| 宿主归属 | **全局特征库**（不建 product/device） | 镜像单位管理定位——先做基础字典层；产品/设备继承留待未来。本期特征可被任意产品引用 |
| 分类维度 | **按特征类型**（property/event/service/relation） | 用户选择；左侧树为四个特征类型节点，与物模型三维结构一一对应 |
| 差异字段存储 | `spec` JSON（参数 / 约束 / 关系语义） | 清单参数复杂且四类各异；JSON 一次成型、改 spec 不改表；DB 层无法约束个别参数由应用层校验器兜底 |
| 参数结构 | 复用阿里云 TSL 数据类型体系（int/float/double/bool/enum/text/date/struct/array） | 与单位管理文档、清单定义完全一致；struct/array 嵌套递归 |
| 关系建模 | `feature_type=relation` + spec 内 `{source, target, type, cardinality}` | 用户要求"关系也是特征"；source/target 指向其它特征或外部实体；本体语义收敛 |
| 单位引用 | property 的 spec.unit 引用 `thingmodel_unit.id` | 打通单位管理预留的 `reference_count` 挂钩点 |
| 前端布局 | Splitter 左右分栏（左=特征类型树/Tab，右=特征列表+动态表单） | 复用单位管理 / dict 范式；动态表单按 feature_type 切换 |

## 与单位管理文档集的关系

```
物模型领域域 (thingmodel)
├── 01–07  单位管理（已落地）        ← 物理量分类 + 单位 + 换算引擎
│          提供：thingmodel_unit 字典
│          预留：unit.reference_count（待特征模块维护）
│
└── 08–14  特征属性管理（本期）       ← 属性/事件/服务/关系 统一特征
           引用：property spec.unit → thingmodel_unit.id
           回填：维护 unit.reference_count 增减
```

本期**不**实现：产品物模型模板（`product`）、设备实例（`device`）、实时数据上报与归一化运行时。这些在特征库稳定后作为下一期独立模块。
