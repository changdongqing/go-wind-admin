# 05 · 前端实现设计（v2.1 · 极简版）

> 仅 React 端。
> **本期采用极简形态：单一 ProTable + Drawer**。
> **kind 仅作为查询表单里的一个下拉过滤项**——不用 Tabs、不用 Splitter、不用 Tree。
> code 自带层级前缀，按 code 升序排序 + 缩进展示即一目了然。

---

## 1. 与 v2.0 的差异

| 维度 | v2.0 | **v2.1（本版）** |
|------|------|------------------|
| 顶部 kind 切换 | Tabs（SYSTEM/SPACE/FACILITY） | **去掉**，kind 进入搜索表单下拉项 |
| 是否需要"当前 kind"状态 | 需要 `useState<Kind>('SYSTEM')` | **不需要**，靠 URL/搜索表单驱动 |
| 默认 kind | 默认 SYSTEM | **不默认**——首次进入显示全量（或仅显示 level=1 提示用户先选 kind） |
| 切换 kind | 点 Tab 触发 reload | 选下拉项 + 点查询/回车（标准 ProTable 行为） |
| 新建大类按钮 | 隐含"为当前 Tab 的 kind 建大类" | 直接弹 Drawer，**用户在 Drawer 里选 kind** |
| 文件数 | 3（index + Drawer + constants） | **3**（同上，更简单） |

## 3. Hooks 层（`api/hooks/category.ts`）

镜像 `unit.ts`，与 v2.0 完全一致：

```typescript
import { useMutation, useQuery, type UseMutationOptions, type UseQueryOptions } from '@tanstack/react-query';
import {
  type categoryservicev1_Category,
  type categoryservicev1_ListCategoryResponse,
  type categoryservicev1_CreateCategoryRequest,
  type categoryservicev1_DeleteCategoryRequest,
} from '@/api/generated/admin/service/v1';
import { makeUpdateMask, type PaginationQuery } from '@/core/transport/rest';
import { queryClient } from '@/core';
import { apiClient } from '@/api/client';

export function useListCategories(q: PaginationQuery, o?: UseQueryOptions<categoryservicev1_ListCategoryResponse, Error>) {
  return useQuery({
    queryKey: ['listCategories', q],
    queryFn: () => apiClient.categoryService.List(q.toRawParams()),
    ...o,
  });
}
export async function fetchListCategories(params: PaginationQuery) {
  return queryClient.fetchQuery({
    queryKey: ['listCategories', params],
    queryFn: () => apiClient.categoryService.List(params.toRawParams()),
    retry: 0,
  });
}

export function useCreateCategory(o?: UseMutationOptions<{}, Error, categoryservicev1_CreateCategoryRequest>) {
  return useMutation({ mutationFn: (data) => apiClient.categoryService.Create(data), ...o });
}

export function useUpdateCategory(o?: UseMutationOptions<{}, Error, { id: number; values: Record<string, any> }>) {
  return useMutation({
    mutationFn: ({ id, values }) =>
      apiClient.categoryService.Update({
        id,
        data: { ...values },
        updateMask: makeUpdateMask(Object.keys(values ?? {})),
      }),
    ...o,
  });
}

export function useDeleteCategory(o?: UseMutationOptions<{}, Error, categoryservicev1_DeleteCategoryRequest>) {
  return useMutation({ mutationFn: (data) => apiClient.categoryService.Delete(data), ...o });
}
```

## 4. 页面（`index.tsx`）

```tsx
import { useRef, useState } from 'react';
import { Button, Popconfirm, Tag, Space, message } from 'antd';
import { ProTable, type ActionType, type ProColumns } from '@ant-design/pro-components';
import { useTranslation } from 'react-i18next';
import ContentContainer from '@/layouts/components/PageContainer/ContentContainer';
import { useProTableScrollY } from '@/hooks';
import { PaginationQuery } from '@/core/transport/rest';
import { fetchListCategories, useDeleteCategory } from '@/api/hooks';
import { useAccess } from '@/access';
import type { categoryservicev1_Category } from '@/api/generated/admin/service/v1';
import CategoryDrawer from './CategoryDrawer';
import { kindValueEnum, levelValueEnum, levelColor, kindColor } from './constants';

const CategoryManagement = () => {
  const { t } = useTranslation('category');
  const [drawer, setDrawer] = useState<{
    open: boolean;
    record?: categoryservicev1_Category;
    parent?: categoryservicev1_Category;
  }>({ open: false });
  const actionRef = useRef<ActionType>();
  const scrollY = useProTableScrollY();
  const access = useAccess();
  const { mutate: del } = useDeleteCategory({
    onSuccess: () => { message.success(t('actions.deleteSuccess')); actionRef.current?.reload(); },
  });

  const columns: ProColumns<categoryservicev1_Category>[] = [
    // ===== 搜索表单第 1 项：kind（核心改动）=====
    {
      title: t('fields.kind'),
      dataIndex: 'kind',
      width: 110,
      valueType: 'select',
      valueEnum: kindValueEnum(t),                 // { SYSTEM: {...}, SPACE: {...}, FACILITY: {...} }
      render: (_, row) => row.kind ? <Tag color={kindColor[row.kind]}>{t(`kind.${row.kind}`)}</Tag> : '-',
    },

    // ===== code（带缩进，直观表达层级）=====
    {
      title: t('fields.code'),
      dataIndex: 'code',
      width: 180,
      fixed: 'left',
      render: (_, row) => {
        const lvl = row.level ?? 1;
        const indent = '  '.repeat(lvl - 1);  // 每级 2 个 nbsp 视觉缩进
        return <span style={{ fontFamily: 'monospace' }}>{indent}{row.code}</span>;
      },
    },

    { title: t('fields.name'), dataIndex: 'name', width: 240, ellipsis: true },

    { title: t('fields.nameEn'), dataIndex: 'nameEn', width: 200, hideInSearch: true, ellipsis: true },

    {
      title: t('fields.level'),
      dataIndex: 'level',
      width: 90,
      valueType: 'select',
      valueEnum: levelValueEnum(t),
      render: (_, row) => <Tag color={levelColor[row.level ?? 1]}>{t(`level.${row.level}`)}</Tag>,
    },

    {
      title: t('fields.isEnabled'),
      dataIndex: 'isEnabled',
      width: 90,
      valueType: 'select',
      valueEnum: {
        true:  { text: t('tags.enabled'),  status: 'Success' },
        false: { text: t('tags.disabled'), status: 'Default' },
      },
    },

    { title: t('fields.referenceCount'), dataIndex: 'referenceCount', width: 100, hideInSearch: true },
    { title: t('fields.sortOrder'),      dataIndex: 'sortOrder',      width: 90,  hideInSearch: true },

    {
      title: t('common.action'),
      key: 'action',
      width: 240,
      fixed: 'right',
      hideInSearch: true,
      render: (_, row) => (
        <Space>
          {access.hasAccessByCodes(['thingmodel:category:create']) && (row.level ?? 1) < 4 && (
            <Button type="link" size="small" onClick={() => setDrawer({ open: true, parent: row })}>
              {t('actions.createChild')}
            </Button>
          )}
          {access.hasAccessByCodes(['thingmodel:category:update']) && (
            <Button type="link" size="small" onClick={() => setDrawer({ open: true, record: row })}>
              {t('actions.edit')}
            </Button>
          )}
          {access.hasAccessByCodes(['thingmodel:category:delete']) && (
            <Popconfirm title={t('tips.deleteConfirm')} onConfirm={() => del({ ids: [row.id!] })}>
              <Button type="link" size="small" danger>{t('actions.delete')}</Button>
            </Popconfirm>
          )}
        </Space>
      ),
    },
  ];

  return (
    <ContentContainer heightMode="fixed" padding="16px" bottomMargin={0}>
      <ProTable<categoryservicev1_Category>
        actionRef={actionRef}
        columns={columns}
        rowKey="id"
        search={{ labelWidth: 'auto', defaultCollapsed: false }}
        scroll={{ y: scrollY, x: 1200 }}
        request={async (params, sort) => {
          // params 中含 kind / level / code / name / isEnabled 等搜索表单值
          const query = new PaginationQuery({
            ...params,
            fieldValues: {
              ...(params.kind  ? { kind: params.kind }   : {}),
              ...(params.level ? { level: params.level } : {}),
              ...(params.code  ? { code_prefix: params.code } : {}),     // code 输入即作为前缀过滤
              ...(params.name  ? { keyword: params.name }     : {}),
              ...(params.isEnabled !== undefined ? { is_enabled: params.isEnabled } : {}),
            },
            // 默认按 code 升序——天然层级顺序
            orderBy: sort && Object.keys(sort).length ? sort : { code: 'asc' },
          });
          const res = await fetchListCategories(query);
          return { data: res.items ?? [], total: Number(res.total ?? 0), success: true };
        }}
        toolBarRender={() =>
          access.hasAccessByCodes(['thingmodel:category:create'])
            ? [
                <Button key="add" type="primary" onClick={() => setDrawer({ open: true })}>
                  {t('actions.create')}
                </Button>,
              ]
            : []
        }
      />
      <CategoryDrawer
        open={drawer.open}
        record={drawer.record}
        parent={drawer.parent}
        onClose={(refresh) => { setDrawer({ open: false }); if (refresh) actionRef.current?.reload(); }}
      />
    </ContentContainer>
  );
};

export default CategoryManagement;
```

### 4.1 视觉效果

搜索表单（一行展开）：

```
[ 种类 ▼ ]  [ 编码 ____ ]  [ 名称 ____ ]  [ 层级 ▼ ]  [ 状态 ▼ ]     [ 查询 ] [ 重置 ]
```

- 种类 下拉：智能系统 / 建筑空间 / 设备设施（选了再查；不选 = 全部）
- 编码：输入 `30`、`3001`、`300101`、`30010100` 任意前缀，立即子树过滤

### 4.2 `CategoryDrawer.tsx`（新建/编辑抽屉）

用 `DrawerForm` + `formRef`。打开方式：

| 入口 | 模式 | `kind` | `parent` | `level` | `code` |
|------|------|--------|----------|---------|--------|
| toolBar「新建」 | 新建（顶层 level=1） | **用户在 Drawer 里下拉选择** | 无 | 1，只读 | 2 位输入，正则校验 |
| 行内「新建子分类」 | 新建（level=parent.level+1） | 继承自父，只读 | 已知，展示 `code · name`，只读 | parent.level+1，只读 | `parent.code + 2 位`，前缀校验 |
| 行内「编辑」 | 编辑 | 只读 | 只读 | 只读 | 只读 |
| 任何模式可编辑 | — | — | — | — | name / nameEn / icon / description / sortOrder / isEnabled |

→ **「kind 切换」这件事现在统一在 Drawer 的"新建顶层分类"入口承担**，而不是页面顶部 Tabs。日常浏览/搜索阶段 kind 是一个普通查询条件，新建顶层分类时才需要"选 kind"。

抽屉表单关键字段（伪示意）：

```tsx
<DrawerForm onFinish={async (values) => { /* ... */ }} initialValues={initial}>
  {/* kind：新建顶层时可选；新建子分类/编辑时只读 */}
  <Form.Item name="kind" label={t('fields.kind')} rules={[{ required: true }]}>
    <Select
      disabled={!!parent || isEditMode}
      options={[
        { value: 'SYSTEM',   label: t('kind.SYSTEM') },
        { value: 'SPACE',    label: t('kind.SPACE') },
        { value: 'FACILITY', label: t('kind.FACILITY') },
      ]}
    />
  </Form.Item>

  <Form.Item label={t('fields.parent')}>
    <Input value={parent ? `${parent.code} · ${parent.name}` : t('tips.topLevel')} disabled />
  </Form.Item>

  <Form.Item name="level" label={t('fields.level')}>
    <InputNumber value={level} disabled />
  </Form.Item>

  <Form.Item
    name="code"
    label={t('fields.code')}
    rules={[
      { required: true },
      { pattern: /^\d+$/, message: t('tips.codeFormat') },
      () => ({
        validator(_, v) {
          const need = level * 2;
          if (v && v.length !== need) return Promise.reject(t('tips.codeLength', { need }));
          if (parent && v && (!v.startsWith(parent.code) || v.length !== parent.code.length + 2))
            return Promise.reject(t('tips.codePrefix', { prefix: parent.code }));
          return Promise.resolve();
        },
      }),
    ]}
  >
    <Input placeholder={parent ? `${parent.code}__` : '__'} disabled={isEditMode} />
  </Form.Item>

  {/* name / nameEn / icon / description / sortOrder / isEnabled */}
</DrawerForm>
```

## 9. 前端注意事项（遵守 react CLAUDE.md）

1. `PaginationQuery` 必须 `new`。
2. 非组件环境（路由守卫等）只用 `fetchListCategories`，不用 `useXxx`。
3. `DrawerForm` 用 `formRef`，无 `useForm`。
4. `ProTable` 的 `scroll.y` 用 `useProTableScrollY` 返回的像素值。
5. 硬编码文本全部走 i18n。
6. `meta.title` 用 `'routes:xxx'` 格式。
7. 禁止手改 `api/generated/`。
8. **默认排序按 `code` 升序**——这是关键，确保层级视觉效果（多 kind 混排时按 kind+code 自动分块）。

## 10. UX 流转图

```
[用户进入分类管理页]
       │
       ▼
[ProTable 展示全部分类（含 SYSTEM/SPACE/FACILITY），按 kind+code 升序自然分块/缩进]
       │
       │  用户操作
       ▼
   ┌────────────────────────────────────────────────────────────┐
   │  搜索栏「种类」选 SYSTEM         → 服务端 where kind=SYSTEM  │
   │  搜索栏「编码」输入 3001         → 服务端 code LIKE '3001%'  │
   │  搜索栏「层级」选 细类           → 服务端 where level=4       │
   │  搜索栏「名称」输入"空调"        → 服务端 name LIKE '%空调%'  │
   │                                                              │
   │  toolBar 「新建」               → Drawer（用户选 kind+建大类）│
   │  行操作 「新建子分类」          → Drawer（kind/parent 自动继承）│
   │  行操作 「编辑」                → Drawer（kind/code 只读）    │
   │  行操作 「删除」                → Popconfirm → DELETE         │
   └────────────────────────────────────────────────────────────┘
```

→ **整个页面只有一个 ProTable + 一个 Drawer**，没有 Tabs、没有 Splitter、没有 Tree。kind 在视图中是个普通过滤项，在 Drawer 里是顶层分类的必填字段——**职责清晰、UI 最简**。

## 11. 与 v2.0 的取舍点对比

| 场景 | v2.0（Tabs） | **v2.1（kind 作查询项）** |
|------|-------------|---------------------------|
| 用户只关心"设备设施"分类 | 直接点 Tab，瞬间切换 | 搜索栏选 FACILITY + 查询（多一次点击） |
| 用户对比看不同 kind 下的分类 | 需要来回切 Tab，记忆负担 | **不切换，列表同时混排，按 kind 分块自然可见** |
| 用户首次进入页面 | 默认 SYSTEM Tab，看不到 SPACE/FACILITY 数据 | **看到全量混排**，对系统结构一目了然 |
| 视觉简洁度 | 一行 Tabs + 一个 ProTable | **只有一个 ProTable** |
| 代码复杂度 | 需维护 activeKind state、Tab 切换重载 | **零额外状态** |
| 「新建大类」按钮语义 | "建当前 Tab 的大类"——隐式依赖 Tab | "建一个大类，kind 在 Drawer 里选"——**显式** |

**结论**：v2.1 在"用户能否快速聚焦某 kind"上略损（多一次下拉点击），在"代码极简、信息全局可见、新建语义显式"上更优。综合 **v2.1 更符合"界面越简洁越好"的原则**。

> 备注：如果未来 kind 数量增长到 6+（DOCUMENT / CERTIFICATE / FORM 等都接入），下拉过滤的优势会进一步放大——Tabs 6 个会很挤，下拉 6 项依然舒展。
